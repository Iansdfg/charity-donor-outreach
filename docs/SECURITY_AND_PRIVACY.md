# Security and Privacy

Minimize collection and retention; use donor IDs in logs and never log raw rows, names, notes, contact details, or gifts unnecessarily. Encrypt production inputs/state, restrict run directories by role, define retention/deletion schedules, and maintain lawful consent records outside this example.

CSV formula prefixes fail validation. All fields are untrusted inert data. Prompt delimiters do not make data trusted; typed fact selection is the primary control. Raw notes never enter system instructions. Suppression precedes provider calls. Jinja autoescape and output checks reject scripts, iframes, event handlers, tracking pixels, placeholders, unauthorized URLs, altered asks, and unsupported claims. Never execute shell text from records. Human review remains mandatory.
